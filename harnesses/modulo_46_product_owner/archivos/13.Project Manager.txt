Hola, qué tal seguimos con la clase de SEO y producto vamos a hablar ahora del rol del project manager. El project manager es el responsable de la correcta ejecución y del desarrollo de un producto. No todos los equipos multidisciplinares tienen esta figura del project manager porque muchas veces está sus funciones que han absorbidas dentro del product corner, pero en caso de que el project manager exista dentro de un equipo multidisciplinar, este es el que se encarga de la operativa y de que la operativa sea exitosa. Este quiere decir que el project manager sería el encargado de gestionar el equipo técnico o, si más no, el responsable de gestionar los recursos del equipo, sean del tipo que sean, sean los perfiles que sean. En caso de tener project manager sería el project manager que sumiría estas funciones y no el product corner. En este caso el product corner pues se vería más liberado de estas tareas de puramente de gestión y podrían focarse en entregar un producto de mayor calidad. Entonces las funciones del project manager quedarían englobadas primero en definir un marco de trabajo para este proyecto. Esto quiere decir pues sentar unas normas claras y concisas con unos objetivos y unos plazos de entrega, es decir el product corner hablaría con el project manager para decir hoy necesitamos esto por este producto que ya está definido tal tal tal tal, vamos a hablar de plazos de entrega cuando debería estar, etcétera. También es cierto que la figura del product corner debería explicar al project manager cómo es todo el proyecto, cómo debería de ser una vez esté entregado y a partir de ahí el project manager sería el encargado de coger el gran proyecto, trocearlo en partes y definir cuándo se va a realizar cada una de las piezas del producto. Entonces esto pasaría a ser ya responsabilidad del project manager. Luego siguiendo lo que estábamos comentando, el project manager tendría dentro de sus funciones el hecho de crear el roadmap y decir primero haremos esto, luego esto, luego esto, luego esto. La visión o las funciones del project manager son puramente de gestión, esto implica que tiene que tener un background un poco más técnico, esto quiere decir que su relación va a estar siempre producto o tecnología, producto o tecnología, producto o tecnología y que el el role del project manager tiene que entender el producto a la la perfección pero también tiene que entender toda la parte de desarrollo y todo lo que implica desarrollar este producto a nivel técnico. Otra de las funciones sería la gestión del equipo de desarrollo, esto quiere decir pues que todos los developers o todas las personas que forman el equipo reportarían directamente al project manager tanto a nivel de gestión de vacaciones como a nivel de one to one, con el nivel de motivación, etc. Sería el project manager que se encargaría de toda esta gestión que como hemos comentado antes en el vídeo de Product Corner es algo que coge bastante tiempo y es algo que en algún punto para aquellos productos que requieren de que el Product Corner o el Project Manager esté muy centrado en investigación, en conceptualización, en partes estratégicas este tiempo de gestión del equipo podrían restar de ahí la figura del project manager que es una figura súper operativa y con una carga de gestión muy elevada. Entonces el project manager dentro de sus funciones tendría a gestionar tanto los individuos de forma individual a los individuos de forma grupal pero también sería el encargado de realizar gestiones con otros equipos, proveedores, etc. Y otra de las funciones que es lo que está aumentando justo ahora sería la de la comunicación tanto con departamentos internos de la casa como con clientes como con proveedores, etc. Entonces en el caso de que exista esta figura el project manager sería el encargado de todas las comunicaciones con cualquier proveedor, con cualquier cliente, con cualquier departamento. Esto también descargaría una gran carga de trabajo al Product Corner o Product Manager que ya se pudieron centrar en crear el mejor producto. Y por último dentro de las funciones del project manager también estaría la parte de QA. Esta es la parte de ver que el trabajo que se ha desarrollado cumple con los requerimientos iniciales del proyecto o que la tarea que se ha realizado ya cumple con los criterios de aceptación o con el definition of don. Esto quiere decir que dentro de las funciones del project manager el revisar, el evaluar, el comprobar que todo funciona a perfección también recae sobre el project manager. Esta es una parte también importante, puesto que todo el tema de revisión también quita mucho trabajo al Product Corner o Product Manager y en caso de que las tareas que deberían de estar superiéndo documentadas con el definition of don bien bien bien especificado con todos los requerimientos bien puestos el project manager podría ser capaz de realizar el QA. También es cierto que hay muchas empresas que este rol de QA lo tienen dentro de la compañía y está integrado dentro de los equipos con lo cual esta pieza o esta responsabilidad podría llegar a no recaer sobre la figura del project manager. Y por último añadir también que esta función puede recaer también y puede ser compartida tanto con el Product Order o Product Manager como con el resto de miembros del equipo. Al final la persona que desarrolla la funcionalidad o la tarea debería ser capaz de revisar si eso que ha hecho está bien o mal porque en la propia tarea debería estar bien descrito bien definido y no debería haber ningún problema en que el propio desarrollador revise y revalide que lo que ha desarrollado funciona la perfección.
