Hola, qué tal? Vamos a ver cuáles son las habilidades que tiene que tener un Project Manager para poder realizar su trabajo en la perfección. Hasta ahora hemos estado comentando los distintos roles de Product Owner, Product Manager y Project Manager. Ahora vamos a ver qué necesita realmente este Project Manager para poder trabajar o con uno o con otro junto a los otros departamentos de la empresa. El primero de ellos es liderazgo. Es cierto que los distintos perfiles, tanto el Product Manager y Product Owner, tienen que tener estas habilidades de liderazgo, más que nada porque al final están desarrollando un producto y tienen que convencer y tener que liderar y tener que hacer que todo el equipo reeme en la misma dirección. El Project Manager, aunque tenga un rol de gestión, también tiene que tener esta parte de liderazgo. Al final el Project Manager, como hemos comentado, dentro de sus funciones está la de gestionar el equipo. Entonces, esto requiere de una serie de habilidades de liderazgo importantes porque el Project Manager, al final, va a ser el que reparte tareas, va a ser el que aprieta el nivel de timings, va a ser un poco pesado en cuanto a que va a ser exigente con el equipo. Esto quiere decir que les va a apretar y esto quiere decir que puede generar fricciones en caso de que no se comuniquen bien las cosas. Entonces, el rol del Project Manager es un rol que tiene que liderar personas. Al margen de liderazgo y obviamente comunicación, que es otro de las habilidades que debería tener un Project Manager, pero que ya son las mismas que tiene un Product Owner o un Project Manager. El Project Manager también debería tener unas habilidades bastante fuertes a nivel de negociación. La negociación es importante, muy importante, por el hecho de que el Project Manager está gran parte de su tiempo negociando. Negunciando con el equipo para ver qué tareas van a hacer, negociando con el equipo, con el Product Owner para plazos de entrega, negociando con el equipo quién va a hacer qué tarea, negociando con el equipo cuándo y con qué prioridades se deberían de abortar cada tarea, negociando con proveedores para saber cuándo o para apretar, para tener antes funcionalidades que van a ser necesarias, negociando con otros equipos para ver cuándo van a poder aportar las cosas que necesita el equipo para poder seguir con su proyecto. Al final, el día a día de un Project Manager es negociar, negociar y negociar. Esto implica tener mano izquierda en el mano derecha, esto implica saber si era certivo para decir que no, o implica tener esta mano izquierda que decía para poder tener más manga ancha y saber cuándo se puede apretar y cuándo no. El hecho de negociar es un básico para un Project Manager y tiene que al final velar por las necesidades del equipo, pero a la vez tiene que ser lo suficientemente inteligente como para saber cuándo y cuándo no, porque al final la motivación del equipo y la motivación de los distintos proveedores o de los distintos equipos también tiene que ir alineada con las necesidades del propio proyecto. Por lo tanto, el Project Manager en este punto juega un rol importantísimo y el hecho de tener estas habilidades de liderazgo comunicativas y de negociación lo hacen que cuanto mejores sean, mejor va a funcionar el proyecto sin ningún tipo de duda.
