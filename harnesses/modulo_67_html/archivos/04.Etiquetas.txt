Los robots de Google cuando analizan nuestras páginas web también detectan si hay enlaces rotos, hay enlaces que generan redirecciones o si hay enlaces que generan algún tipo de error y en el caso de que los haya pues Google no lo puntúa positivamente. Los principales códigos de error bueno los principales códigos son los siguientes. El código 200 que quiere decir que la respuesta es correcta y que no hay ningún problema. El código 301 que de esta forma estamos indicando a Google que nuestra página está movida de forma permanente que le hemos cambiado de URL, le hemos cambiado de ubicación o cualquier cosa de este tipo. Si yo tengo una página web y tengo un enlace y quiero que esta URL cambie por el motivo que sea es importante que yo no pierda toda la gente externa que está apuntando a este enlace. Por lo tanto yo configuraré el servidor web para que todo el mundo que apunte todos los enlaces que apunten a mi URL antigua se redireccionen a mi URL nueva. Esto sería correcto crear redirecciones 301 es una práctica correcta. Lo que no sería correcto es que si desde mi página web, desde mi propia página estoy enlazando a este enlace antiguo pues lo deje así. El usuario sí que iría visitando al enlace antiguo se generaría esta redirección y sí que vería el contenido nuevo pero le estaría haciendo saltar de forma innecesaria. Es mucho mejor que yo corrija los enlaces internos de mi página web y ponga directamente el enlace definitivo el enlace actualizado para que esta redirección no se genere. De esta forma Google verá que no tenemos redirecciones en la página que son saltos innecesarios y además verá que actualizamos el nuestro contenido que siempre lo tenemos al día sin este tipo de errores.
