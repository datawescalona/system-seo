En el segundo punto de este curso hablaremos más en detalle de cómo se compone la estructura de una página HTML. El lenguaje HTML dispone de unos elementos fundamentales para conseguir una buena estructura de página. Esta estructura bien formada facilitará a los navegadores y a los robots la lectura de esta página. Y es muy importante facilitar esta lectura para que los robots vean cómo se compone la estructura de una página HTML. Bueno, los robots vean que esta página está bien estructurada, que los contenidos tienen sentido y que nos la puntúen bien y la tengan en buena consideración. El contenedor principal de una página HTML es la etiqueta, la propia etiqueta HTML. Y también comentar que para crear o editar una página HTML únicamente necesitamos un editor de texto y guardar este archivo con el nombre que queramos del archivo y la extensión punto HTML. Explica un poco la imagen que tenemos, en la que se define básicamente la estructura de una página HTML. Veremos lo que comentamos anteriormente. El elemento principal está dentro de la etiqueta HTML. El elemento principal es la etiqueta HTML con todo el contenido de la página dentro. Y veremos que los siguientes elementos principales son el HEAD y el BODY. Es común cuando estás programando en HTML utilizar las tabulaciones, separar estas etiquetas para que se vea claramente de forma visual que etiqueta está dentro de las otras. Entonces, en esta imagen también hemos intentado que visualmente se vea cuál es la etiqueta principal, cuál es la etiqueta padre y cuáles son las etiquetas que están dentro. Dentro de la etiqueta HEAD normalmente incluimos metaitiquetas, las veremos posteriormente, pero podemos tener información del autor, la descripción, el título de la página, etc. También se incluyen enlaces, enlaces a hojas de estilos, enlaces a páginas de script que son externas. Y también podemos encontrar normalmente la parte donde utilizamos para incluir contenedores de información. El más famoso o el más utilizado es Google Tag Manager. Entonces también el código de Google Tag Manager que veremos posteriormente está incluido en la parte del HEAD y otros scripts de seguimiento, piques de conversión, etc. La parte del HEAD es totalmente invisible para el usuario. El contenido que tiene no es contenido visual, es contenido utilizado con otro fin. Y entramos en la etiqueta de body. La etiqueta de body sí que contiene toda la parte visual, todo el contenido que el usuario, bueno todo el contenido que se muestra en pantalla cuando el navegador carga la página. Y básicamente una buena estructura de HTML debería de contener un elemento header, en el que normalmente no se incluimos el logo, un buscador, algunos enlaces, un título, los títulos principales de las páginas están dentro de la etiqueta H1. También normalmente suele incluir o debemos incluir para que se considere que es una estructura óptima una etiqueta de navegación, el elemento estructural nav, que básicamente se incluye en enlaces de navegación por la página web, un menú. Vemos otro elemento estructural que es el main, importante que el main sólo aparezca una vez por página y el main sí que va a contener, bueno va a ser el contenedor del resto de textos y contenidos de la página. Y tenemos otro elemento que es el asign, que bueno en esta imagen está representado lateralmente, no tiene por qué ser siempre su ubicación, lo podemos incluir de muchas formas, pero bueno es una barra complementaria que no contiene el contenido principal de la página, tiene otro tipo de contenidos más puntuales, eventos, algún banner publicitario, etcétera, etcétera. Dentro del main podemos encontrar muchos elementos, a modo de ejemplo aquí tenemos section, article, y dentro de cada uno de ellos pues subtítulos, textos, imágenes, etcétera, section y article no son elementos estructurales, están aquí dentro del main a modo de ejemplo y el último elemento estructural que tenemos por ver es el footer, que es el pie de página.
