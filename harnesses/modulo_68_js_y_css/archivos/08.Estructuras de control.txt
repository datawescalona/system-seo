Hola, bienvenidos al punto 7 del curso de JavaScript del Master de SEW de Bixero, donde veremos las principales estructuras de control del lenguaje de JavaScript. Las estructuras de control de flujo permiten alterar el flujo de ejecución del programa. Es posible conseguir que un código se ejecute o no, se repita múltiples veces, deje de ejecutarse o se ejecute únicamente si detecta un error. En el vídeo incorpora las principales estructuras de control que se pueden encontrar también en la mayoría de lenguajes de programación. Cuando nosotros ejecutamos un código, normalmente la ejecución es secuencial, va de la línea 1 hasta la última. Pero las estructuras de control de flujo, lo que nos permiten es alterar este orden y como comentábamos anteriormente, nos permiten saltarnos alguna parte, dirigir la ejecución del código al sitio que nos interesa, etc. Una de las principales estructuras de control de flujo de los lenguajes de programación, y en este caso en concreto del JavaScript, son las sentencias if y else. Esta estructura de control permite validar una expresión buleana y, en base al resultado si es true or false, seguir con la ejecución de un código u otro. En caso de que el resultado sea false, el código a ejecutar es el que está dentro de la sentencia else. Comentar que la sentencia else no es obligatoria, es opcional, podemos utilizar un if sin un else y se pueden anidar varias sentencias if else dentro del mismo código. A cada valor o case le indicaremos el código ejecutar en caso de coincidencia. Después de ejecutar un código se ejecutará el siguiente case si no se encuentra la sentencia break y se pueden añadir un case default que se ejecuta cuando no se encuentra ninguna coincidencia. Y el tercer bucle que existe en casi todos los lenguajes de programación es el do-while. Este bucle permite definir un bloque de código ejecutar mientras se cumpla la condición. Y por último, podemos analizar la estructura de control de try cat y finally. Esta estructura nos permite programar acciones a realizar si se produce algún error durante la ejecución de un código. En la parte del try se contiene el código de evaluar y el código que puede producir estos errores. En la parte del cat se incluyen las acciones a realizar cuando alguno de estos errores se ha producido. Y en la parte de finally, que es una parte opcional, no es necesaria para que funcione el try cat, pero se incluyen las acciones que queremos realizar siempre, si hayan producido o no estos errores.
