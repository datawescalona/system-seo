Bienvenidos al octavo punto del apartado de JavaScript del Master Deseo de Vixeo, donde vamos a ver, es un punto muy cortito, muy simple, las diferentes formas que tenemos de mostrar mensajes al usuario utilizando este lenguaje. Por una parte, la forma más fácil de mostrar mensajes en pantalla es lanzando una ventana en el gente con la siguiente sintaxis, sería poner comando alert, incluyendo entre comillas el texto que queremos mostrar y esto el resultado sería lo que vemos a la derecha del código, mensaje a mostrar al usuario y el botoncito a aceptar. Esta forma de mostrar información es la menos recomendable, por un lado es muy molesta y poco usable y por otro lado detiene la ejecución del código JavaScript y detiene la carga de la página web hasta que el usuario no la acepta. Si queremos mostrar mensajes de depuración no visibles para los usuarios en general, se pueden utilizar los mensajes de consola, con las sintaxis de console.loc y entre paréntesis y entre comillas el mensaje a hacer. Esta forma es la más recomendable a utilizar mientras se programa y se necesita mostrar información de forma rápida para conocer el estado del programa. Estos mensajes sólo se muestran en las herramientas de desarrollo de los navegadores y esto debería de ser únicamente una herramienta de desarrollo y cuando esté el código finalizado y lo queramos subir a producción al entorno real, donde este código se tiene que ejecutar, deberíamos eliminar todos estos mensajes. En la captura de pantalla vemos las herramientas de desarrollador de Google Chrome en la parte de consola y vemos que el mensaje que se ha puesto en código se reproduce. Esto es muy útil si estamos haciendo un bucle complejo o nos está fallando algo y no sabemos bien dónde ir añadiendo mensajes de este tipo pues antes de entrar en el bucle y aquí podemos añadir el valor de x en este punto es 4. Dentro del bucle el valor de x es el que sea y de esta forma podemos ir enviando los mensajes a nosotros mismos para saber en qué estado o hasta dónde llegan o dónde entran la ejecución de los códigos y ir devugando los valores concretos que tiene el código en esos puntos. Son bastante útiles sobre todo para desarrollo pero no son mensajes pensados en mostrarse al usuario y otra forma de comunicarnos desde JavaScript por el usuario es mostrar estos mensajes en el HTML es la forma más compleja y recomendable de mostrar mensajes al usuario de hecho es la que hemos estado utilizando en los ejemplos anteriores y recalcamos que para ello necesitamos asegurar que el código HTML es el elemento que queremos utilizar se ha cargado previamente no podemos ejecutar un script pidiéndole que modifique el contenido de un elemento de HTML si el código HTML no se ha cargado en la página y también para usar esta forma necesitamos conocer una manera de acceder a este elemento ya sea con el selector del ID o con cualquier otro tipo de selector pero tenemos que tener capacidad de seleccionarnos de saber identificar para modificar este mensaje y bueno con esto acabamos la parte de mostrar mensajes al usuario y entraremos en el siguiente punto del curso en el que hablaremos qué formas tenemos para pedirle datos al usuario
