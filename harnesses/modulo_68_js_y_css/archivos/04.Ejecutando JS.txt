Hola, bienvenidos de nuevo al curso de JavaScript del Master Desay de Vixeo, en el que empezaremos con el tercer punto de este curso, donde hablaremos de cómo se ejecuta el código JavaScript en un navegador. El objetivo de JavaScript de este lenguaje es sobre todo el facilitar la interacción web del usuario con la página web. Por lo tanto, estamos hablando de mejorar la usabilidad y accesibilidad de la misma. Por ello, todos los códigos JavaScript siempre van a ir vinculados a un documento HTML. No podemos ejecutar nada en JavaScript, sino tenemos una página HTML. Un documento HTML puede tener vinculados varios archivos JavaScript a la vez o varios scripts no siempre tienen que ir en archivo, como veremos más adelante, pueden estar también incrustados en el mismo código HTML. Pero bueno, la idea es que un documento HTML puede tener varios códigos JavaScript y un archivo JavaScript puede estar enlazado también desde varios documentos HTML. Entre HTML y JavaScript estamos hablando de una relación de varios a varios. Cualquier HTML puede tener infinitos JavaScript enlazados y un mismo documento JavaScript puede estar enlazado desde varios HTML. Al ser un lenguaje cliente, el usuario siempre tendrá acceso a ver a inspeccionar el código JavaScript que se esté ejecutando. Desde cualquier página web con cualquier navegador que disponga de unas herramientas básicas de programación que hoy en día casi todos los navegadores las tienen. Por ejemplo, en el navegador de Google Chrome, sin cualquier página web hacemos botón derecho inspeccional, veremos que se nos ve directamente el código HTML y el código CSS de estas páginas. Pues bien, con estas herramientas también podemos analizar el código JavaScript de cualquier página web. Por lo tanto, tenemos que tener en cuenta que no tenemos que pasar la seguridad de nuestra página web en el código JavaScript. Por ejemplo, si nosotros tuviésemos una validación de contraseñas escrita en JavaScript, sería muy fácil para cualquier usuario que sepa un poco de programación con el inspeccionador del navegador encontrar esta contraseña y estaríamos hablando de algo inseguro. En el caso de la validación de campos pasa algo similar. Sí que de cara a mejorar, como estamos hablando siempre, la usabilidad del usuario y la velocidad. Es fácil que podamos hacer las validaciones de los campos cuando se están escribiendo en JavaScript. Muchas veces, cuando no sé, por ejemplo, estamos rellenando un campo de texto y nos piden un correo electrónico y no lo introducimos en un formato válido que contenga una roba y que contenga un dominio con un punto, un punto es, punto como, etcétera. Muchas páginas webs pueden hacer que este mismo campo, cuando lo hemos acabado de escribir y hemos cambiado al siguiente campo del formulario, se nos marque de color rojo, nos haga una alerta, nos avise de que hay algo incorrecto en este campo. Todas estas variaciones sí que se hacen con JavaScript. Están pensadas para facilitar al usuario el relleno de este formulario. A nivel de programación, como es un lenguaje en torno cliente, todos los scripts y todos los codios HTML se han descargado en el navegador y son alterables de cara al usuario. Un usuario puede desactivar esta validación, puede desactivar todos estos mensajes que el navegador muestra buscando la parte del código de JavaScript que hace que se produzca y alterándolo y modificándolo. Por lo tanto, sería posible que si únicamente hacemos una validación a nivel de JavaScript, el usuario consigue ese enviar información a nuestro servidor, a nuestra base de datos, no valida con correos electrónicos, no valido. Entonces es importante que se hagan las dos validaciones cuando se está programando. Es importante validar que cuando el usuario está rellenando un formulario, todo lo que pueda validarse antes de darle a enviar mejor, porque nos evitamos que el servidor detecte que está mal, que la petición vuelva hacia atrás y que el usuario tenga que volverlo a enviar después de corregirlo. JavaScript nos ayuda a que el usuario cuando le dé a enviar tenga claro y esté seguro de que todos estos campos están bien, pero cuando estamos hablando de seguridad de la página web y que no nos introduzcan datos que no queramos, a pesar de que se pasan todas las validaciones de JavaScript, tenemos que tener en cuenta que estas validaciones las puede alterar a un usuario y tenemos que volver a realizar todas estas validaciones a nivel de servidor, que el servidor es quien nos garantizará que estos datos son correctos y decidirá guardarlos o rechazarlos en base a los criterios. Pero muy importante, repito, no se usa JavaScript para mejorar la seguridad de la página web, aunque en algunos casos lo utilicemos para validar campos y para encontrar errores, que un número de teléfono tenga x dígitos o que un correo electrónico tenga la composición correcta, la sintaxis correcta de los correos electrónicos, todas las validaciones se tienen que volver a ejecutar a nivel de servidor, porque si no no podemos garantizar que el usuario no las haya modificado o alterado. ¿Cómo integramos un código JavaScript dentro de un HTML? Pues tenemos tres maneras muy sencillas. La primera de ellas es la que vemos aquí en pantalla, consiste en el gato de la página, introducir las etiquetas de script e insertar dentro el código JavaScript. Si seguimos con este ejemplo, veremos que es una forma muy sencilla, pero no podemos reutilizar este código para otra página. Es importante saberlo, si nosotros hacemos un código JavaScript en un archivo externo, si que lo podemos llamar desde varias páginas web, desde varios enlaces. Pero si incrustamos dentro de encabezado de una página web un código HTML, pues obviamente este no lo podremos reutilizar ni lo podremos usar desde otra página. Aunque lo habitual es que las etiquetas de script estén en el fea de una página web, es importante tener en cuenta que pueden estar en cualquier sitio. Si las ponemos en el body, en el footer, en cualquier parte del HTML, el código JavaScript funcionará de la misma forma. Únicamente tenemos que tener en cuenta que si estamos alterando, modificando o actuando sobre elementos HTML, es importante garantizar que estos elementos estén cargados, que esta parte del código haya cargado, siempre antes de que nosotros ejecutemos el script. En el caso de que ejecutemos un script y no garanticemos que si este script utiliza una parte, un elemento del HTML, este elemento no se ha cargado previamente, pues puede fallar. Para esto hay otras técnicas, hay eventos de javascript que puedes hacer, que un código te cargue cuando ha acabado de cargar toda la parte de HTML, etcétera, etcétera. Es fácil de controlar. La segunda forma es lo que comentamos anteriormente, tener un documento aparte con la extensión .js de JavaScript y enlazarlo de la siguiente forma. También lo podríamos enlazar desde cualquier parte del documento. Habitualmente utilizamos el geal para hacer esto y bueno, simplemente añadiendo dentro de las etiquetas script el atributo src con el valor, la ruta del archivo JavaScript que queramos enlazar. Esto ya nos quedaría enlazado y esta página utilizaría este código de la misma forma que si lo hubiéramos hecho como en el ejemplo anterior introduciendo todo el código directamente dentro de estas etiquetas. Esta es la forma más común y nos permite además enlazar este documento que hemos desarrollado desde diferentes páginas web.
