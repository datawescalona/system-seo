Primero de todo comentar que es un lenguaje case sensitive. Esto significa que si nosotros declaramos una variable en mayúscula, tanto una letra en mayúscula como toda en mayúscula como cualquier combinación de mayúsculas y minúsculas, esta misma combinación la tenemos que respetar a lo largo de todo el código, porque las mayúsculas son diferentes a las minúsculas en este lenguaje y si en otro sitio escribimos esta misma variable de otra forma, sin la mayúscula o lo que sea, el código no la encontrará, y generará un error y no funcionará. También comentar que cada una de las sentencias de programación de este lenguaje se pueden separar o con el signo punto y coma o con saltos de línea. ¿Qué pasa? Normalmente estos códigos JavaScript luego se comprimen para subirlos a la web y que ocupen lo mínimo posible. Y dentro de estas compresiones una cosa muy habitual es la de eliminar saltos de línea. Entonces no es nada recomendable programar teniendo en cuenta que los saltos de línea finalizan sentencias y siempre se debería de introducir puntos y comas al finalizar los comandos de este lenguaje. En nombre de las variables hay que iniciar con una letra o con una barra baja, no pueden iniciar con números, y se recomienda utilizar las barrabajas únicamente para los constructores, reservarlas para esto. Un método constructor es un método especial para crear e iniciar un objeto creado a partir de una clase. Entonces por norma general iniciamos las variables en JavaScript únicamente por letras. Las variables compuestas se suelen escribir utilizando la convención low camel case, que consiste en concatenar todas las palabras de esta variable compuesta usando mayúsculas en las primeras letras de cada una de estas palabras. Declaración de variables. La declaración de variables en JavaScript es muy simple. Únicamente tenemos que introducir las palabras let obar y a la derecha el nombre de la variable que le queramos dar. JavaScript es un lenguaje de programación no tipado. Eso significa que cuando nosotros declaramos una variable a diferencia de otros lenguajes de programación en JavaScript no es necesario especificar el tipo de dato que se va a almacenar. Con las palabras let obar declaramos variables que pueden ser numéricas, que pueden ser cadenas de texto, que pueden ser arrays, etcétera. Además la misma variable puede almacenar varios tipos de datos durante la ejecución del programa. Esto quiere decir que una variable que no ha guardado un número en la misma ejecución del programa sin hacer ningún cambio en la declaración de esta variable, yo puedo guardar una letra o puedo cambiar el tipo de dato que se almacena en esta variable. Las declaraciones con var y let no son exactamente lo mismo, son diferentes. Las variables declaradas con la palabra let se limitan al bloque de código en las que están declaradas. No se pueden usar fuera de este mismo bloque de código. En cambio las variables declaradas con var son globales y se pueden utilizar en cualquier parte del documento. Una variable declarada con let únicamente se puede declarar una vez y en cambio con var podríamos declarar la misma variable varias veces que esto no nos produciría ningún error. La palabra let es más restrictiva, aporta más seguridad y es más común utilizarla por estos motivos.
