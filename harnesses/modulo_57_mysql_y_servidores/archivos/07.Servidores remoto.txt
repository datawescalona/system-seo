Hola, bienvenidos al siguiente punto del curso de servidores del Master Deseo de Vixeo en el que trataremos las aplicaciones de servidores remotos. El poder conectarse remotamente a un equipo hoy en día es muy importante ya que muchas veces decidimos trabajar con servicios en la nube y no tener nosotros que montar físicamente todas las máquinas. Por eso este punto es muy importante y espero que os parezca interesante. Como hemos dicho anteriormente, hoy en día es muy común instalar servidores en la nube y no tener acceso presencial a estas máquinas. Al no tener acceso presencial no podemos conectar un monitor, un teclado, un ratón para poder gestionar y utilizar estas máquinas y por lo tanto lo primero que tenemos que hacer es garantizar que podemos establecer una conexión remota con las mismas. La forma más recomendada de establecer esta conexión cuando no se necesita en torno gráfico es utilizar el protocolo SSH. Para utilizar este protocolo como vemos a la izquierda tenemos que realizar la instalación de la aplicación servidor como por ejemplo la aplicación OpenSSH server. En esta página web vemos varios ejemplos de instalación en sistemas Ubuntu Debian y el otro ejemplo en un sistema Centros of Fedora. Son diferentes distribuciones según el sistema operativo que estemos utilizando como servidor el servidor SSH será una aplicación u otra y se instalará de una forma u otra. Siempre tenemos que mirar cómo se hace en el sistema operativo que estamos trabajando. Una vez instalado este servicio vemos que hay que ejecutar un segundo comando que lo que hace es reiniciar este servicio. Esto es necesario en el caso de querer aplicar cambios. Normalmente lo que hacemos es instalar el servicio, configurar todos los parámetros necesarios para que este servicio funcione como nosotros queremos. Por ejemplo, el caso SSH como es una conexión remota que da acceso a toda la máquina se suelen configurar IPs de entrada. Solo se permiten conexiones que vienen de una IP determinada para que solo desde una oficina en concreto se puedan conectar. Una vez tenemos configuradas y establecidas todas estas opciones procedemos a reiniciar el servicio para que se apliquen estos cambios. Y a partir de aquí podemos utilizar este servicio. Como hemos dicho anteriormente también por un lado instalamos la aplicación servidor que nos permite hacer todo esto. Y por otro lado utilizamos una aplicación cliente para establecer la conexión. En este caso hay una aplicación que se llama PUTI que es súper común, que es muy común y muy utilizada para establecer conexiones SSH. Hay que tener en cuenta que el protocolo SSH únicamente te devuelve o únicamente permite ver la consola de comandos de la máquina servidor. En ningún caso podremos ver en torno gráfico con estas dos aplicaciones. Si necesitamos un poquito más de interacción y no trabajar tanto por consola de comandos podemos instalar otras aplicaciones. Un ejemplo muy conocido y es una aplicación muy utilizada es este que vemos aquí en pantalla que se llama Webmin. Al final esta aplicación lo que hace es instalar otro servidor web en esta máquina con ciertas opciones para que puedas tener un acceso vía web al administración de este equipo. Vale tanto para instalar aplicaciones, para configurar algunos servicios, para ver como vemos en esta segunda captura de pantalla los recursos que se están utilizando en la máquina, cuanto disco disponible tiene, cuanta memoria RAM, etcétera. Al final esto nos facilita la administración de la máquina es más visual acceder a esta interfaz web que acceder a una consola de comandos. Pero bueno también tenemos que tener en cuenta que esto es un servidor. Contra más cosas instalemos más recursos estamos usando al final si yo quiero un servidor web pues intentaré que todos los recursos se destinen a esta parte y en un servidor contra menos cosas se instalen mejor por temas de seguridad por temas de actualizaciones por temas de rendimiento contra menos cosas tengamos instaladas mucho mejor el criterio de un servidor es que tenga las menos cosas posibles. Entonces en este caso podríamos decir que este tipo de herramientas por un lado sí que facilita la gestión pero por otra ensucian un poquito y perjudican un poco al rendimiento de la máquina. Si os fijáis lo que hace esta herramienta es abrir un nuevo servidor web y para tener dos servicios web abiertos de forma permanente en la misma máquina no sé si se ve bien la barra del navegador vale pero estamos accediendo a localhost 2.10.000 lo que está haciendo esta herramienta es instalar un servidor web y en vez de usar el puerto 80 por defecto lo han configurado con el puerto 10.000 para no entrar en conflicto con los servidores que puedan tener también instalados un servidor web que esté ya a la escucha por este puerto. Si el sistema operativo dispone en torno gráfico es posible utilizar herramientas de escritor de remoto por ejemplo si utilizamos windows Windows Server incorpora ya opciones para permitir escritor de remoto a sus equipos servidores y los equipos clientes de Windows las distribuciones que no son Windows Server que son Windows 7, Windows 10 o los Windows que vengan por delante los Windows posteriores todos tienen herramientas de conexión escritor de remoto que hacen referencia a la captura de pantalla que hay más a la izquierda es la conexión escritor de remoto del propio Windows que te permite conectarte a servidores de Windows pero bueno también existen otras aplicaciones algunas con versiones gratuitas otras de pago como estas de aquí como Team Beaver, como VNC, como NIDESC, etc. todas estas aplicaciones tienen un comportamiento similar a todo lo que estamos viendo a nivel de servidores las tres tienen una aplicación servidor que hay que instalar en la máquina a la cual nos queremos conectar que funcionarán con un puerto por defecto de cada una de las aplicaciones entiendo que podremos cambiar este puerto a nuestro antojo por si nosotros en nuestros servidores el puerto lo estamos usando para algo en concreto seguramente nos den opciones de cambiar este puerto por otro y en las máquinas clientes desde las que nos queramos conectar pues tenemos que instalar otra aplicación que no será la misma, será una aplicación cliente que nos permitirá conectarnos a esta máquina y poder acceder a todo su escritorio.
