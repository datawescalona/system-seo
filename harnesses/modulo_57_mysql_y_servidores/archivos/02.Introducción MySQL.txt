Hola, bienvenidos al primer punto del curso de MySQL, del Master Deseo de Vixeo, en el que veremos una pequeña introducción a qué son las bases de datos. Bien, una base de datos es un sistema en el que se almacena una serie de datos de forma grupada y estructurada, y con relaciones entre ellos. Los datos de una base de datos son utilizados por los sistemas de información de empresas y particulares. Nacen para resolver el acceso a la gran demanda de datos de hoy en día, es una tecnología rápida y sencilla, y es una tecnología de representación tabular, que quiere decir que se representa en formato de tablas. Antiguamente, tal y como vemos en la imagen, la información se clasificaba de forma física y se necesitaban estanterías y se necesitaban almacenes para guardar mucha información, las empresas lo tenían todo en físico en papel. Esto cambia y hoy en día este gran volumen de información que antiguamente se almacenaba de forma física se almacena en las bases de datos. Por eso necesitamos que sea una tecnología rápida y de acceso sencillo, ya que de esta forma podemos hacer consultas desde páginas webs y cargar datos rápidamente, además de tener almacenada una gran cantidad de información. Como hemos dicho anteriormente, dentro de una base de datos lo que vamos a encontrar son tablas. Cada tabla representa una entidad estructurada en filas y columnas. A cada fila se le llama instancia y a cada columna un atributo. Como vemos en este ejemplo, la tabla o entidad persona contiene diferentes atributos, como pueden ser el nombre, apellido, edad, etc. y cada uno de los registros que se llenen dentro de esta tabla, se llamarán instancias, cada una de las filas representará la instancia 1 o la instancia 2, que serán estos datos rellenados en base a la información de cada una de las personas que introduzcamos en esta tabla. Dentro de los sistemas gestores de las bases de datos podemos definir el esquema de las bases de datos, incluyendo una descripción de las tablas y de los campos, nombres de usuario, privilegios, roles, podemos controlar qué usuario consulta qué datos o qué tablas. También podemos asignar permisos de por ejemplo, división, modificación, podemos decidir que un usuario únicamente puede hacer alta, así que no puede eliminar. Podemos configurar todo tipo de permisos. También el esquema de una base de datos incluye información de la ubicación del almacenamiento de la misma y también podemos ver y consultar estadísticas del uso de esta base de datos. Aquí vemos un ejemplo, una pequeña captura de pantalla de una aplicación gestora de bases de datos que se llama MySQL orbent, en la cual estamos abriendo una tabla que se llama film y vemos dentro de esta tabla de una forma muy visual que columnas contiene, qué tipo de datos almacena cada una de estas columnas y los diferentes atributos que se le pueden asignar a estos campos. Por ejemplo, si es clave primaria, si es un valor que no puede ser nulo, si es un valor que tiene que ser único, etcétera.
